# Phase 4 — Path C Cloud Deploy: Planning & Decisions

## Overview
Path C separates **local (full data)** from **cloud (summary only)**. The local Mac app stays unchanged. A second Streamlit Cloud instance reads `public/summary.json` — a lightweight JSON file regenerated by an automated local script every 4–6 hours during market hours.

**Components to build:**
1. `scripts/refresh.py` — local Mac script: fetch prices, recompute, generate summary, push to GitHub
2. `public/summary.json` generator — logic to extract allocations, ratios, tickers, prices from local DB
3. `launchd` plist — macOS scheduler to run refresh.py on a schedule
4. `app/main.py` toggle — `IM_DATA_SOURCE=local|cloud` env flag
5. Cloud deployment — second Streamlit Cloud instance reading JSON only
6. Install docs — how to set up launchd + run refresh manually

### 4. Cloud Password & Pre-Auth View

**Question:** How should this work in practice?

- Streamlit Cloud's built-in `streamlit_authenticator` or simple password check

### 5. Cloud vs. Local Codebase
**Current plan:** Single codebase, toggle via `IM_DATA_SOURCE` env var.

```python

if data_source == "cloud":
    holdings = load_holdings_from_json("public/summary.json")
else:
    holdings = load_holdings_from_db()  # local
```

**Question:** Should the cloud instance have fewer tabs/views (e.g., hide Settings, Leverage)?

### 6. GitHub Push Credentials
**Current draft:** `refresh.py` commits + pushes `public/summary.json` to the private repo.

**Question:** How should refresh.py authenticate with GitHub?

**Option A: SSH key (recommended for CLI)**
- Generate SSH key, add to GitHub
- refresh.py uses `git push` (no explicit auth needed)
- More secure; works offline

**Option B: Personal Access Token (PAT)**
- Create GitHub PAT with `repo` scope
- Store in macOS Keychain or `.env` (not in repo)
- refresh.py uses `git clone --https` with token

**Your choice:**
- [ ] Option A (SSH key)
- [ ] Option B (PAT)

### B. Streamlit Cloud Setup
1. Log in to [share.streamlit.io](https://share.streamlit.io)
2. Create **new app** (second deployment):
   - Repo: `Segey-P/investments-monitor`
   - Branch: `main`
   - File: `app/main.py`
   - Name: `investments-monitor-public` (or similar)
3. Set **Secrets** (Settings → Secrets):
   ```
   IM_DATA_SOURCE = "cloud"
   IM_PASSWORD = "your-cloud-password"  # if using custom auth
   ```
4. Add password protection via Streamlit Cloud settings

### C. Local Testing (before launchd install)
1. Test `refresh.py` manually:

### D. launchd Install
Once refresh.py is working:
1. Copy template plist to `~/Library/LaunchAgents/`
2. Load it:
   ```bash
   launchctl load ~/Library/LaunchAgents/com.sergey.investments-monitor.plist
   ```
