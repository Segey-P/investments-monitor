# Integrated Investment & Asset Monitor: Master Specification

## 1. Executive Summary

The Integrated Investment & Asset Monitor is a personal portfolio-monitoring system for a single Canadian investor. It consolidates registered and unregistered investment accounts plus cryptocurrency into a unified analytics dashboard, with special attention to **HELOC leverage** as the differentiating feature. Real estate and mortgage balances are tracked for a holistic net-worth view. Cash is tracked as individual holdings (ticker='cash') per account on import, eliminating separate aggregate accounting. **All data remains local only** — the app runs on macOS with SQLite, accessed via Claude Code CLI for daily analysis routines and email summaries.

## 2. Core Objectives

- **Centralization:** Aggregate holdings across RRSP, TFSA, Unregistered, and Crypto accounts.
- **Leverage Oversight:** Track HELOC utilization and its impact on portfolio risk. Tax-deductibility calculations are **out of scope**; the user handles tax treatment externally.
- **Net Worth Analysis:** Incorporate property value and mortgage balance for a household view.
- **Privacy-First (Path C):** No dollar totals in the cloud. Tickers and proportions are acceptable exposures.
- **Claude-Native:** Local SQLite is readable by Claude Code (CLI) on the user's Mac for daily analysis routines.

## 3. High-Level Architecture

| Component | v0.1 (this spec) | v1 target | Role |
|---|---|---|---|
| **UI** | Streamlit | FastAPI + React | Dashboard and forms |
| **Backend** | Python (same process as Streamlit) | FastAPI | Business logic |
| **Storage** | SQLite (`data/portfolio.db`, `.gitignored`) | SQLite | Holdings, HELOC draws, snapshots, settings |
| **Price engine** | `yfinance` with 15-min cache | unchanged or Polygon | Equity/ETF/crypto quotes |
| **FX** | Bank of Canada daily rate (cached) | unchanged | USD → CAD conversion |
| **Scheduler** | macOS `launchd` | unchanged | Price refresh + public summary regen every 4–6h during market hours |
| **Auth** | Streamlit password gate + session timeout | FastAPI session auth | Protects full-data view |
| **Cloud deploy** | Streamlit Community Cloud, password-gated, reads `public/summary.json` only | TBD | Remote summary view |

## 4. Security & Data Integrity

### 4.1 Privacy model — Path C (decided)

- **Local (full):** `data/portfolio.db` contains all holdings, quantities, ACB, HELOC draws, mortgage balance, property value. Never leaves the Mac. Never committed.
- **Cloud (summary only):** `public/summary.json` is regenerated by the scheduler and committed to the private repo. Streamlit Community Cloud serves it behind a password.
- **Local encryption:** Skipped for v0.1. macOS FileVault provides at-rest encryption. Revisit if threat model changes.

### 4.2 Privacy model — Local-only (finalized)

**All data stays on the Mac.** No cloud deployment, no public summaries, no remote access.

- `data/portfolio.db` is the single source of truth
- SQLite is `.gitignored` (never committed)
- Claude Code CLI accesses the DB directly for daily routines
- Streamlit app (local only) reads full data
- Email summaries generated by Claude Code from the local DB

### 4.3 Authentication & session

- **Local app:** Password gate on launch. Password stored as hash (bcrypt) in `data/portfolio.db` settings table.
- **Cloud app:** Streamlit Cloud native password (separate from local password).
- **Session timeout:** Default 15 minutes of inactivity. Warning banner at 60 seconds remaining with a "Stay signed in" button that extends the session. On expiry: redirect to login; no data rendered.
- **Pre-auth view:** Login screen shows the same cloud-safe summary (proportions + leverage ratio) so the user gets value even before unlocking.

### 4.4 Manual overrides

Override support is limited to **manually entered** data:

| Category | Source | Overridable? |
|---|---|---|
| Holdings (ticker, qty, ACB) | Broker CSV import | No — fix at source (re-import) |
| Market prices | yfinance | No — fix via refresh |
| FX rate | BOC daily | No |
| HELOC balance, rate, limit | Manual | Yes |
| Cash balance (aggregate) | Manual | Yes |
| Real estate value | Manual | Yes |
| Mortgage balance, rate, renewal | Manual | Yes |
| Other assets / other debt | Manual | Yes |

## 5. Claude-native integration

The app is designed to coexist with Claude Code daily routines on the user's Mac.

- Local Claude Code (`claude` CLI) reads `data/portfolio.db` directly via `sqlite3` shell. No app-level encryption ensures this works.
- Claude Code web (e.g., `claude.ai/code`) operates on the git repo only — it sees code, specs, and the public summary. Code changes can be developed from the web; end-to-end testing with real data must happen locally.
- Future option: expose a read-only MCP server over the SQLite file if Claude needs structured access. Not required for v0.1.

## 6. Stack migration path (v0.1 → v1)

- **v0.1:** Streamlit. Goal is to prove the data model, import pipeline, and calculation correctness. UI fidelity is secondary.
- **v1:** Re-platform to FastAPI + React only if Streamlit's layout ceiling blocks the V2 "Dark Command Strip" aesthetic. Decision gate: after Phase 4 (Path C deploy).

## 7. Deferred / out of scope

Documented for completeness; not part of v0.1.

- Multi-profile / Family / Parents' portfolios (design leaves `portfolio_id` in the data model for future)
- Transaction log + tax-lot editing + cost-basis detail drawer (ACB is entered once, per-position)
- Sector concentration analytics
- HELOC tax-deductibility calculations, per-drawdown flags, marginal tax rate inputs
- Leverage ratio historical sparkline (trend chart)
- FX rate history and per-ticker currency toggle
- Mobile-optimized layout
- App-level encryption at rest (SQLCipher)
- Onboarding flow
